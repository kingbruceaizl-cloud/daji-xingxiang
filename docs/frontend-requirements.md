# 前端开发需求

## 1. 设计原则

大吉形象是工作型产品。公开首页承担品牌识别和产品入口作用，登录后首屏直接进入可操作的形象设计工作台。

界面风格应简洁、清晰、适合反复使用，重点突出项目、客户素材、商品选择和生成结果。

品牌风格参考大吉形象官网，统一使用官网 LOGO、品牌红、真人服务场景图和清晰的大标题。详细规则见 `/docs/brand-style-guide.md`。

公开首页可以使用沉浸式图片首屏，但必须保留“开始形象设计”的直接入口，并在首屏底部露出下一段产品内容。后台、项目和工作台仍采用紧凑的工作型布局。

## 2. MVP 信息架构

### 第一阶段

- 本地 Mock 模式无需登录即可创建本地项目并体验形象大师流程。
- 线上真实模式采用邀请制团队账号；公开注册页显示邀请说明，不提供自行注册入口。
- 支持负责人、管理员、员工三类角色，只有负责人和管理员显示后台入口。
- 登录后工作台只展示中文账号摘要，不展示原始认证凭据或接口返回数据。

### 核心页面

- 工作台首页
- 项目列表
- 形象大师流程
- 素材与生成结果
- 后台管理

## 3. 形象大师工作流

页面：`/studio/[projectId]`

没有 Supabase 时，`/projects/new` 会创建本地项目并跳转到 `/studio/local-...`；也可以使用 `/studio/demo` 进入演示工作台。

建议采用步骤式布局：

1. 客户素材
2. 风格选择
3. 商品搭配
4. 生成图片
5. 生成视频
6. 保存与导出

### 客户素材

- 支持上传图片；未配置 Supabase 时使用本地预览兜底，上传后立即替换客户素材预览。
- 支持上传视频，第一阶段可先存储，不必完成视频理解。
- 真实模型模式下，上传客户图片后可调用 Doubao-Seed-2.1-Pro 生成结构化图片分析；分析结果需要展示人物特征、可保留特征、搭配建议和风险提示，并允许操作者确认或补充。
- “图片转文字”在前端统一呈现为可编辑的中文分析结果，不显示模型原始响应或内部字段。
- 上传控件需要在选择文件后先校验文件类型和大小，并用中文提示不支持的文件或超大文件。
- 显示素材预览、上传状态、移除按钮；移除上传素材后恢复演示客户素材，并同步生成面板的输入素材。

### 风格选择

- 展示后台配置的风格模板。
- 每个风格显示名称、封面图、简短描述和标签。

### 商品搭配

- 按分类选择商品：发型、妆造、衣服、裤子、鞋子、手饰/饰品等。
- 支持真实商品和素材型商品。
- 商品卡片显示图片、名称、标签、是否真实商品。
- 已选商品固定展示在底部确认栏，支持移除和清空选择。
- 商品和风格选择变化后，需要同步更新生成提示词和生成任务参数。

### 生成图片

- 先由 Doubao-Seed-2.1-Pro 根据客户图片分析、风格和商品选择生成结构化方案，再生成生图提示词。
- 展示方案摘要和生图提示词摘要，允许操作者确认并手动补充。
- 支持手动补充提示词。
- 默认使用“后台自动路由”；1.0 生产默认路由到 Doubao-Seedream-5.0 完整版，调试时仍可选择其他已启用模型通道。
- 发起生成任务后展示状态：排队中、生成中、成功、失败、已取消。
- 真实模型任务创建后自动查询 `/api/jobs/:id`，成功后展示转存到素材库的结果预览。
- 成功后展示结果图列表，支持保存到项目和下载结果。
- 生成结果、项目详情和后台任务列表必须展示中文模型通道、模型名称、任务类型和状态，不直接暴露内部英文标识。
- MVP 本地模式使用演示模型通道，点击“生成形象图”即可返回演示结果并显示任务状态。

### 生成视频

- 基于选中的形象图片发起视频生成。
- 在工作台中选择视频模板、脚本文案和音乐，由 Doubao-Seed-2.1-Pro 生成视频提示词和镜头方案，再同步到视频生成任务参数。
- 1.0 生产默认路由到 Doubao-Seedance-2.0。
- 第一阶段只做短视频任务，不做复杂长视频时间轴。
- MVP 本地模式继续返回演示视频封面和任务信息；真实模式由 Doubao-Seedance-2.0 返回结果，并在转存 Supabase Storage 后展示系统素材地址。

## 4. 后台管理页面

### 商品库

- 商品列表
- 新增/编辑商品
- 上传商品图
- 配置商品提示词描述
- 标记真实商品或素材
- 上架/下架

### 风格模板

- 风格名称
- 风格封面
- 风格标签
- 正向提示词片段
- 反向提示词片段

### 视频模板

- 模板名称
- 视频比例
- 时长
- 运镜说明
- 变装方式
- 默认脚本

### 音乐库

- 音乐名称
- 文件上传
- 情绪标签
- 适用场景

### 模型配置

- 模型通道
- 模型名称
- 能力类型
- 任务能力路由
- 是否启用
- 默认参数

密钥不在前端展示明文。

### 团队权限

- 负责人和管理员可通过邮箱邀请员工。
- 负责人可调整员工角色，系统不允许取消最后一位负责人。
- 负责人和管理员可设置每位员工的月度文字、图片、视频额度、同时任务数和真实生成开关。
- 员工可看到自己的生成结果和清晰的额度不足提示，不能访问后台管理页面。

火山方舟在前端统一显示为“火山方舟（豆包）”，模型显示名称使用 Doubao-Seed-2.1-Pro、Doubao-Seedream-5.0 完整版和 Doubao-Seedance-2.0；完整 API 模型 ID 不在普通工作台暴露，只在后台配置与诊断信息中展示。

工作台默认使用“后台自动路由”，根据当前行动内容选择模型。操作者仍可手动切换演示通道或指定通道，方便上线前调试。

后台新增或编辑配置时，需要先展示中文字段说明，再展示结构化配置模板，避免只让运营人员面对内部字段。

## 5. 未来画布预留

MVP 不开发画布页，但数据结构和导航可以预留“创作画布”入口为禁用状态。后续再引入 React Flow，实现项目素材、生成任务和结果之间的节点关系。
