# 大吉形象 PRD

## 1. 产品概述

大吉形象是一款 AI 形象设计产品，帮助形象顾问、运营人员或商家基于客户上传的图片/视频，快速完成发型、妆容、服装、鞋包、饰品等搭配，并生成可交付的形象图片和变装短视频。

MVP 第一阶段不做 Lovart 式复杂画布，而是做“形象大师无画布版”：用步骤式流程降低使用门槛，让用户按顺序上传素材、选择风格和商品、生成结果。

## 2. 产品目标

- 让非技术人员可以完成客户形象设计。
- 将商品库和 AI 生图/视频能力打通。
- 支持真实商品和搭配素材两种商品类型。
- 后台可配置模型、商品、脚本、视频模板、音乐。
- 为未来 Lovart 风格画布版预留项目、素材、任务和模型抽象。

## 3. MVP 范围

### 本期要做

- 本地 Mock 模式无需登录即可体验形象大师流程；线上真实模式必须使用团队账号登录。
- 采用邀请制注册，支持负责人、管理员、员工三类角色。
- 形象设计项目创建。
- 客户图片/视频上传。
- 商品库选择：发型、妆造、衣服、裤子、鞋子、手饰/饰品等。
- 风格模板选择。
- 客户图片理解、形象分析和结构化方案生成。
- 生图提示词和短视频提示词生成。
- AI 形象图片生成任务。
- AI 变装短视频生成任务。
- 生成任务状态查看。
- 生成结果保存到素材库。
- 后台管理商品、分类、风格模板、视频脚本、音乐和模型配置。
- 后台邀请员工、调整角色，并设置员工的月度文字/图片/视频额度与并发任务上限。

### 本期不做

- Lovart 复杂画布。
- React Flow 节点编排。
- 支付、积分、会员和面向客户的计费系统。
- 客户授权确认流程。
- 真人肖像授权确认流程。
- 完整长视频剪辑器。

## 4. 用户角色

### 项目负责人

系统首位账号自动成为项目负责人。可管理模型、商品、模板、脚本、音乐、生成任务、团队角色和员工额度；系统必须始终保留至少一位负责人。

### 管理员

由负责人任命。可管理业务后台、邀请员工和配置员工额度，但不能变更员工角色或取消最后一位负责人。

### 形象顾问

系统中的“员工”角色。可为客户创建形象项目、上传素材、选择商品、生成图片和视频；不能访问管理后台。真实模型生成受个人月度额度、并发上限和启用状态约束。

### 客户

后续可选角色。第一阶段不直接登录系统，只作为被设计对象。

## 5. 核心流程

### 形象设计流程

1. 创建形象项目。
2. 上传客户正面照、半身照、全身照或视频。
3. 选择目标风格，例如通勤、约会、轻奢、国风、韩系、商务、婚礼、短视频种草等。
4. 选择商品和素材，包括发型、妆容、上衣、裤子、裙装、鞋子、包、饰品。
5. Doubao-Seed-2.1-Pro 分析客户图片、目标风格和商品信息，生成结构化形象方案。
6. Doubao-Seed-2.1-Pro 根据确认后的方案生成并优化生图提示词。
7. Doubao-Seedream-5.0 完整版发起文生图或图生图任务。
8. 选择满意图片后，由 Doubao-Seed-2.1-Pro 生成短视频提示词和镜头方案。
9. Doubao-Seedance-2.0 发起变装短视频任务。
10. 保存、下载或进入后续长视频扩展位。

### 后台配置流程

1. 配置 AI 模型通道和模型。
2. 创建商品分类。
3. 上传商品图、素材图或真实商品。
4. 配置风格模板和提示词片段。
5. 配置视频脚本模板。
6. 配置音乐。
7. 查看生成任务和失败原因。
8. 邀请团队员工并配置角色、月度用量与同时任务数。

## 8. AI 模型策略

AI 应用 1.0 已确认使用火山方舟豆包模型作为默认生产方案，同时保留统一模型通道抽象，不把业务写死到某一家供应商。

### 1.0 默认模型分工

- `Doubao-Seed-2.1-Pro`：文字分析与输出、形象方案、文案、提示词、分镜、图片理解和图片转文字。
- `Doubao-Seedream-5.0` 完整版：文生图、图生图、多参考图生成和模型支持范围内的图片编辑。
- `Doubao-Seedance-2.0`：文生视频、图生视频和人物变装短视频。

内部模型通道标识使用 `volcengine`，中文展示为“火山方舟（豆包）”。实际 API 模型 ID 使用火山方舟控制台当前账号可用的完整、带版本 ID，并通过服务端环境变量配置，不在业务代码中硬编码。

### 预留模型通道

- OpenAI：预留文字、生图和多模态能力。
- 即梦：预留图像和视频生成能力。
- 可灵：预留视频生成能力。
- 通义：预留文字、图像和视频生成能力。

模型能力按任务类型归类：

- 文字生成：形象方案、提示词、脚本文案和商品描述。
- 图片理解：客户照片、商品图和参考图识别。
- 文生图
- 图生图
- 图生视频
- 视频生成
- 长视频合成
- 文案生成

后台需要提供“任务能力路由”配置，让不同操作默认调用不同模型：

- 生成形象方案和脚本：走文字生成模型。
- 分析客户图片和商品图片：走图片理解模型。
- 未上传客户素材时生成图片：走文生图模型。
- 基于客户照片生成形象图：走图生图模型。
- 基于形象图生成变装短视频：走图生视频模型。
- 长视频后续扩展：走长视频生成模型。

1.0 默认路由为：

- `text_generation`、`image_understanding` → Doubao-Seed-2.1-Pro。
- `text_to_image`、`image_to_image` → Doubao-Seedream-5.0 完整版。
- `image_to_video`、`video_generation` → Doubao-Seedance-2.0。
- `long_video_generation` → 暂不配置生产模型，继续使用扩展位或演示通道。

`image_understanding` 在 1.0 同时覆盖图片语义描述与图片中可见文字提取，暂不单独引入 OCR Provider。

### 1.0 验收标准

- 文字分析、形象方案、生图提示词和视频提示词可从服务端真实生成。
- 客户图片可完成语义理解和图片转文字，并返回可编辑的结构化中文结果。
- 文生图、图生图和基于形象图的变装短视频至少各完成一次真实联调。
- 任务状态、模型 ID、请求参数、错误和输出素材关系可追踪。
- 图片和视频结果已转存 Supabase Storage，不长期依赖第三方临时 URL。
- 浏览器、前端代码和日志中不出现火山方舟密钥。
- 真实任务创建前由数据库原子检查个人启用状态、月度额度和并发上限，Mock 演示任务不计费。
- 失败或长时间未更新的真实任务自动形成系统告警，恢复成功后自动关闭；后台可确认、关闭或重新打开告警。

## 9. 商品库策略

商品分两种：

- 真实可售商品：有价格、库存、购买链接、商品编码。
- 搭配素材：只用于形象生成，没有真实售卖信息。

商品都应支持：

- 分类
- 标签
- 主图
- 参考图
- 提示词描述
- 适用风格
- 是否上架

## 10. 当前开放问题

- 长视频样片和参考风格暂未确定。
- 火山方舟账号中三个模型的完整模型 ID、可用地域、配额和实际请求参数需要在开发接入前从控制台核准。
- Seedance 2.0 使用客户真人素材时是否必须经过火山方舟可信素材库、真人认证并取得 Asset ID，需要用目标账号核准；项目暂不自建授权确认页面，但不能绕过供应商强制的合规流程。
- 商品库第一批分类和字段可在后台雏形完成后继续细化。
- 是否需要下载水印、品牌片头片尾，后续再确认。
